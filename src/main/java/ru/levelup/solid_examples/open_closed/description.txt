Open/Closed: программные сущности должны быть открытыми для расширения, но закрытыми для модификации.

Закрытыми для модификации означает: единственная причина, по которой вы можете менять код класса / функции / модуля -
это непосредственно изменение заложенной в него функции или фикс ошибок работы этой функции. Других причин быть не может.
Открыты для расширения означает: если вам надо, чтобы ваш класс / функция / модуль могли выполнять заложенные функции в
новом окружении - они должны это поддерживать без изменения их кода.

Пример: у нас есть сортировщик. Допустим, что изначально перед нами стоит задача сортировать целочисленные массивы - мы
пишем для этого метод. Со временем возникает требование, что неплохо бы сортировать еще и строки - хорошо, добавляем
метод сортировки строк. В дальнейшем изменяются требования / настоение / погода, и метод для сортировки целых чисел
становится неактуален, а вот сортировать объекты какого-либо класса было бы очень кстати. Что ж, идем в сортировщик,
удаляем ненужный метод, добавляем нужный, живя в ожидании, что это непоследнее его изменение.
Решение: написать универсальный сортировщик, который будет принимать любые объекты, и жить спокойно.
